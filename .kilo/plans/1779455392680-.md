# Plan: Add Avg Price and Date Range Filters

## Goal

Extend the existing search/sort/filter system in the AI Model Browser with two new from-to range filters:

1. **Average price range** — filter models whose weighted avg price (per 1M tokens) falls between two user-supplied bounds.
2. **Date range** — filter models whose `created` date falls between two user-supplied dates.

These additions must coexist with existing filters (search, provider, free-only, sort) and with URL persistence.

## User-Confirmed Design Decisions

- **UI placement**: Collapsible "More filters" panel below the existing single-row of controls.
- **Date inputs**: HTML5 native date pickers (`<input type="date">`, YYYY-MM-DD).
- **Price inputs**: Number inputs with `$/1M tokens` units, matching what is displayed on `ModelCard`.

## Detailed Implementation

### 1. State and types in `ModelsBrowser.tsx`

Add four new state variables (all stored as strings to mirror the input value and avoid NaN flicker when the field is empty):

```ts
const [priceMin, setPriceMin] = useState(searchParams.get("priceMin") || "");
const [priceMax, setPriceMax] = useState(searchParams.get("priceMax") || "");
const [dateFrom, setDateFrom] = useState(searchParams.get("dateFrom") || "");
const [dateTo,   setDateTo]   = useState(searchParams.get("dateTo")   || "");
```

Empty string == "no bound on this side". Inputs whose value is an empty string are treated as no-op.

### 2. URL persistence

Extend `updateUrl()`:

- Accept `priceMin`, `priceMax`, `dateFrom`, `dateTo` in the `next` partial.
- Append params only when non-empty: `?priceMin=…&priceMax=…&dateFrom=YYYY-MM-DD&dateTo=YYYY-MM-DD`.

Extend the `searchParams` resync `useEffect` to read these four params and call the four setters.

`handleReset` must also clear all four new state vars (before the `router.replace("/")`).

### 3. Filter logic in `filteredModels` memo

Compute parsed bounds once at the top of the memo:

```ts
const minPriceNum = priceMin === "" ? null : Number(priceMin);
const maxPriceNum = priceMax === "" ? null : Number(priceMax);
const fromTs      = dateFrom === "" ? null : Date.parse(dateFrom) / 1000; // seconds
const toTs        = dateTo   === "" ? null : (Date.parse(dateTo)   + 86_399_999) / 1000; // include the entire "to" day
```

Notes:
- Inclusive both ends. The "to" date is shifted to end-of-day so `dateTo=2024-01-15` includes models created any time on Jan 15.
- `Number.isFinite` checks guard against malformed input — invalid bounds are treated as "no bound" (null).
- Negative or NaN price input is coerced to null with a `Number.isFinite(...) && x >= 0` check.

Inside the existing `models.filter(...)` callback, add three checks alongside the existing ones:

```ts
// Avg price (in $/1M tokens to match display unit)
const avgPerMillion = calculateAveragePrice({
  input: parseFloat(model.pricing.prompt),
  output: parseFloat(model.pricing.completion),
  cacheRead: model.pricing.input_cache_read != null
    ? parseFloat(model.pricing.input_cache_read)
    : null,
}) * 1_000_000;

const matchesPriceMin = minPriceNum == null || avgPerMillion >= minPriceNum;
const matchesPriceMax = maxPriceNum == null || avgPerMillion <= maxPriceNum;

// Date range — model.created is a Unix timestamp (seconds).
// Models with `created === 0` are excluded when ANY date bound is active
// (they represent meta/router models with no real date — see context.md).
const dateActive = fromTs != null || toTs != null;
const matchesDate =
  !dateActive ||
  (model.created !== 0 &&
    (fromTs == null || model.created >= fromTs) &&
    (toTs   == null || model.created <= toTs));

return matchesSearch && matchesProvider && matchesFree
       && matchesPriceMin && matchesPriceMax && matchesDate;
```

The avg-price calculation is hoisted into a small helper inside the memo to avoid duplicating logic between the filter and the existing price-sort path. Both paths can call the same `getAvgPricePerMillion(model)` lambda.

### 6. Behavior summary

| Scenario | Behavior |
|----------|----------|
| Both price bounds empty | No price filter applied |
| Only `priceMin` set | Show models with avg ≥ priceMin |
| Only `priceMax` set | Show models with avg ≤ priceMax |
| Both set, min > max | Empty list, inline warning under inputs |
| Negative/NaN price input | Treated as "no bound" |
| Both date bounds empty | No date filter applied |
| Only `dateFrom` set | Models created on/after that date (excludes `created === 0`) |
| Only `dateTo` set | Models created on/before end-of-day for that date (excludes `created === 0`) |
| Free-only + price range | Both apply; free models only match if `0` falls in price range |
| URL: `?priceMin=0.5&priceMax=5&dateFrom=2024-01-01` | All read on mount, panel auto-expands |
| Reset button | Clears all four new fields plus existing filters |
